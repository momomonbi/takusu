FOUNDATION — 基 本 方 針

この手引きの立ち位置

本題に入る前に、この手引きが何をして、何をしないかを明確にしておく。
以下の三つの前提を、お互いに認識した上で読み進めてほしい。

PRINCIPLE I

プライバシーは
「権利」である

見せない・知らせないことは、信頼の欠如ではない。個人として当然に保障されるべき権利で、相手への感情とは独立して存在する。この区別を、お互いに認識しておく。

MOST IMPORTANT
PRINCIPLE II — 最重要

実務に必要なのは
「存在」だけ

相続の実務は、口座の存在さえ把握できれば銀行・証券会社側で進められる。中身の開示は一切必要ない。

この事実が、この手引き全体の土台。ここを理解していれば、「全部見る／見せる」は必要ないと分かる。

PRINCIPLE III

技術的に
境界線を引ける

iPhoneの仕組みを使えば、見せたい情報と見せたくない情報を技術的に分離できる。感情論や約束ではなく、システム上の仕組みで境界線を保証する。これが最も確実な方法。

以 下 、 本 文
— 前 書 き —

この手引きの目的は、お互いのプライバシーを守りながら、遺された側が実務で困らない状態をつくること。そのために必要なのは、感情や約束ではなく、仕組みと手順。

配偶者に対して「全部知りたい」という気持ちが生まれるのは、人間として自然なこと。同時に、「見せたくない領域がある」というのも、同じように自然なこと。どちらが正しくて、どちらが間違っているという話ではない。

この手引きが採る立場は、「どちらの感情も認めた上で、実務的にどう折り合いをつけるか」。見せない領域があっても相続は実行できる。見せたい領域は最小限でも遺族は困らない。その最適解を、仕組みとして組み立てることに焦点を絞る。

お互いに、この手引きの前提を受け入れられるなら、以下のページを実装していってほしい。
受け入れられない部分があれば、それは二人で話し合うしかない領域。

ADVICE — 読 み 始 め る 前 に

二人へのアドバイス

01

一気に読まなくていい

この手引きは長い。一度に全部理解しようとせず、気になるところから読めばいい。順番に読む必要もない。何度も行き来するのが普通。

02

二人が理解することが理想

仕組みを動かすには、お互いがこの手引きの考え方を知っておく必要がある。片方だけが理解していても、もう片方が設定していないと機能しない。相手にも要点だけは伝えておくのがベスト。

03

まずは「相互登録」だけでも

この手引きの中で最も効果が大きいのは、お互いのiPhoneに「故人アカウント管理連絡先」を登録すること。他の全項目をやらなくても、これだけで相続時のハードルは劇的に下がる。今日、二人でこれだけは完了させる価値がある。

04

相手の分類に干渉しない

何を「個人用」にしたか、相手に聞かないこと。聞かないのがルールで、そうすることで仕組みが成立する。これは信頼の話ではなく、設計の話。

05

気持ちは置いていける

「全部知りたい」「見られたくない」——どちらの気持ちも、この仕組みの中では問題にならない。技術で境界が引かれるので、感情で悩む必要がない。これが最大の利点。

THE CLASSIFICATION — 分 類 の 原 則

一人の持ち物を、
"生活の柱" と "個人用" に分ける

旦那もりぴも、それぞれ複数の口座・サービスを持っている。
その一つ一つを「生活の柱」か「個人用」かに分類する。
同じ一人の中に、共有すべきものと守るべきものが、共存する。

VS
SHARE — 共有する

"生活の柱"

死後すぐ、遺された側の生活に直結する口座・契約。存在を知らないと遺族が路頭に迷うもの。

  • 家計の引落し口座
  • 住宅ローン・家の契約
  • 生命保険・医療保険
  • 給与振込口座
  • 共同管理している貯蓄
  • 公共料金・通信費の支払い元
共有するのは「存在」まで。残高や詳細は見せなくていい。「どの銀行に、何の目的の口座があるか」だけ伝わればOK。
KEEP — 守るもの

"個人用"

生前は存在すら共有しなくていい。相続実務には影響せず、必要なら死後に銀行照会等で発見される。

  • 個人用NISA・副業口座
  • 趣味のサブスク契約
  • 個人用クレジットカード
  • SNS・個人的なメッセージ
  • 買い物履歴・閲覧履歴
  • 見せたくない写真・メモ
触れないのが最善。相続実務上も、個人領域に踏み込む必要はない。ほふり（証券保管振替機構）やマイナンバー照会で後から必要な口座は発見できる仕組みがある。

お互いに、同じ分類を自分の中で行う。
それぞれの「生活の柱」だけをリスト化して相手に渡す。「個人用」は手元に残す。
この二つをはっきり分けるのが、この仕組みの核心。

PRIVACY SHIELDS — プライバシーを守る技術

お互いの領域を
技 術 的 に 守 る 方 法

「見せたくない／見られたくない」と言うと重たく聞こえるが、そうじゃない。
そもそも見れない構造を最初から作っておく。感情の問題にしないための、技術的な設計。

ロック付きメモ
iPhone標準「メモ」アプリはメモ単位でパスワードを設定できる。ロック付きメモは故人アカウント管理連絡先でも中身が見えない。個人的な記録・ID情報はここに入れる。

設定方法：メモを開く → 共有ボタン → 「ロック」→ パスワード設定。本体Face IDとは別の独立パスワード。
MEMO LOCK
アプリ別 iCloud同期OFF
設定 → 名前 → iCloud → 「すべてを表示」から、アプリごとにiCloud同期をOFFにできる。同期OFFのデータは故人アカウント管理連絡先からはアクセス不可。

例：メモの一部カテゴリだけ同期OFF、写真のみ同期OFFなど粒度を選べる。
iCloud CONTROL
写真の「非表示」＋「ロック」
写真アプリで画像を「非表示」アルバムに移動できる。iOS 18以降は非表示アルバム自体にFace ID / パスコードロックがかかる。見せたくない画像はここへ。

さらに iCloud写真の同期をOFFにすれば、Apple経由でもアクセス不可。
PHOTO HIDE
パスワードマネージャの分離
1Password / Bitwarden などは「Vault（金庫）」を複数作れる。家族共有用Vaultと個人用Vaultを分離し、共有用のみ相手と共有。個人用は独立マスターパスワード。

iPhoneの「パスワード」アプリ（iCloudキーチェーン）は故人アカウント管理連絡先でもアクセス不可なので、そのまま使うのも手。
VAULT SPLIT
「相手へ」メモ専用の運用
標準メモアプリに「相手へ」というタイトルのメモを1つだけ作る。iCloud同期ON、ロックなし。書くのは「生活の柱」の存在情報と本体パスコードだけ。

それ以外の一般メモは、別フォルダに入れて同期OFFまたはロック付きに。この明確な分離が運用の要。
FOR YOU
スクリーンタイム制限（応用）
特定のアプリにスクリーンタイムでパスコードをかけられる。個人用のアプリを1つずつロックすることで、本体が開いても個別アプリには入れない層を作れる。

SNSアプリ、個人用銀行アプリなどに有効。
APP LOCK

重要 — プライバシー層は「多層」で使う

一つの仕組みに頼ると抜け道ができる。例：iCloud同期はOFFでも、iPhone本体が開けば見える。逆に本体を開けないなら、iCloud同期ONのものは故人アカウント管理連絡先から見える。

重要な個人情報は「ロック付きメモ」＋「iCloud同期OFF」のような組み合わせで二重に守る。お互いに、自分の領域を技術的に守る責任がある。

🏦 銀行アプリ
中難度
  1. アプリを開いてみる。生体認証要求が出たら、「パスワードでログイン」に切替。
  2. パスワードリセットを選択 → 登録メール or SMS宛にリセットリンク送信。
  3. iPhoneに届くSMS / メールが見えるので、そこから新パスワードを設定してログイン。
  4. ログイン後は残高確認のみ。送金・解約はしない。正規の相続手続きが必要。
  5. 同時並行で銀行の相続窓口にも連絡し、正規ルートで解約・移管を進める。
📈 証券アプリ
最優先
  1. 銀行アプリと同じ要領でパスワードリセットを試みる。
  2. 取引パスワード（別設定）が必要な場合あり。これもリセットできる証券会社が多い。
  3. ログイン後、保有銘柄・評価額のスクショを撮っておく（相続税申告の資料になる）
  4. ⚠ 最重要：信用取引・FX・先物（借金して取引するタイプ）が有効かを確認。あれば最優先でポジションを止める——詳細は次のセクション。
  5. 証券会社に電話し、名義人死亡の届出 → 口座停止 → 相続手続きへ。
🔐 SMS / Email 2FA
低難度
  1. 本体が開いていれば、SMSは受信・閲覧可能。SIMカードが生きている限り有効。
  2. Gmail等も本体から開けるので、メールの認証コードも読める。
  3. SIMが使えない場合は別のiPhoneにSIMを挿し替えてSMS受信する方法もある。
  4. キャリアの契約者名義変更が先行すると電話番号が使えなくなる可能性——変更前に必要な認証は済ませておく。
🔑 Authenticator アプリ
高難度
Authenticator（オーセンティケーター）とは 2段階認証で6桁のコードを表示するスマホアプリのこと。銀行・証券・SNSなどのログイン時に、パスワードに加えてこの6桁コードを入力する仕組み（=認証システム）。Google Authenticator、Microsoft Authenticator などが代表的。
  1. Google / Microsoft Authenticator などのアプリは本体にしか入っていない。本体が開けば使える。
  2. 本体が開けない場合は突破不可。各サービスの「バックアップコード」（印刷保管してあるはず）か、相続窓口からの再登録しかない。
  3. バックアップコードは生前に印刷して金庫に入れるのが理想。
  4. 基本的にはこのアプリに依存しすぎないように、生前に設計しておく。
💬 LINE・メッセージ
中難度
  1. LINEは本体が開けばログイン状態のまま使える。連絡先・グループ一覧が見られる。
  2. 訃報の連絡先特定には便利。結局トーク内容も目に入るので、ショックを受ける情報が出てくる可能性がある覚悟はしておく。
  3. LINE Pay等の残高は正式な相続手続きが必要（LINE公式ヘルプ参照）
  4. トーク履歴の引き継ぎは原則不可。端末初期化で全て失われる。残したい会話は見つけたタイミングでスクショ。
🔁 サブスク解約
低難度
  1. Apple設定 → 名前 → サブスクリプションでiTunes経由の契約が一覧。ここは簡単に解約できる。
  2. Google Playの場合はPlayストア → アカウント → 定期購入。
  3. アプリ外契約（Spotify公式、Netflix、各種有料サイト等）は各サービスのWebから解約。メール履歴で契約先を確認。
  4. クレカ明細・通帳の引き落とし履歴で定期引き落としを全洗い出し → 1つずつ止める。
  5. 重要：解約画面が見つからない／契約先が分からない引き落としでも、泣き寝入りしない。引き落とし明細の会社名で検索して連絡先を調べ、電話やお問い合わせフォームから「契約者死亡のため解約希望」と直接連絡する。多くの会社は遡って解約・返金対応してくれる。
  6. 引き落としが続いてしまった場合も、気付いた時点で連絡すれば、過去分の返金対応をしてくれる会社もある（業者次第）。諦めず連絡を。

なぜ、これだけは別格に危険なのか

例えば、買い物で考えると
ふつうの株（現物株）
買ったバッグと同じ

10万円でバッグを買ったら、最悪でも10万円が消えるだけ。そのバッグが流行遅れで5万円になっても、損は5万円で止まる。それ以上の請求は来ない。

←→
信用取引・FX・先物
お金を借りて買う

10万円預けて、100万円分の取引をする。うまくいけば儲けも10倍、でも失敗すれば預けた10万円を超えて、借金を返さなきゃいけない。どれだけ借金が膨らむかは相場次第。

一番怖いのは、持ち主が亡くなって操作できなくなった後も、市場は動き続けること。誰も止める人がいないまま、時間とともに借金が膨らんでいく。気付いた時には、預けた金額の何倍もの損失になっていた——という事例が実際にある。

信用取引・FX・先物は、「借金をして取引する仕組み」。
これが、普通の株（現物株）との決定的な違い。

ふつうの株（現物株）なら、株価が下がっても持っていた金額の範囲で損失が止まる。預けたお金が0円になることはあっても、それ以上は請求されない。

でも信用取引・FX・先物は違う。相場が悪い方に動くと、預けたお金を超えて借金が発生する。本人が操作できない状態で市場が動き続けると、損失が時間とともに膨らんでいく。

実際の報告事例：遺族が気付いた時には、預けていたお金を大幅に超える借金が発生していたケースがある。他のどの項目よりも、これが最優先の理由。

ACTION 01
証券会社を特定する
iPhoneアプリ一覧（SBI証券・楽天証券・マネックス・松井など）、メール検索「取引報告」「約定」、通帳の証券会社への振込履歴。どこで取引していたかを1秒でも早く突き止める。
ACTION 02
証券会社へ電話
「名義人が亡くなった。信用取引やFXで持っているポジションがあるか確認したい」と伝える。配偶者であることを証明すれば、状況を教えてくれる。電話番号は公式サイトで「相続手続き」と検索。
ACTION 03
ポジションの決済を依頼
損失が膨らむのを止めるため、「反対売買」（持っているポジションを処分する操作）を証券会社に依頼できる場合がある。相続手続きが全部終わる前でも、これで損失拡大は止められる。
ACTION 04
相続放棄の検討
もし損失が他の資産（預金・家など）の合計を超えそうなら、相続放棄（死亡を知ってから3ヶ月以内）という選択肢がある。一人で判断せず、税理士か司法書士に即相談。

生前にできる最低限の配慮：株取引をしているなら、証券会社の名前だけは相手に伝わるようにしておく（「相手へ」メモに書くだけでいい）。銘柄・残高・建玉内容は秘密のままで大丈夫。遺族は連絡先さえ知っていれば、あとは証券会社側が全部処理してくれる。

  • 「相手へ」メモを作成（iCloud同期ON・ロックなし） 書くのは「生活の柱」のみ：iPhoneパスコード／家計関連の金融機関名／家族保険／緊急連絡先／株取引（特に信用取引・FX）をしている場合は証券会社名。残高・個人用口座は書かない。
  • 個人用のメモ・情報は「ロック付きメモ」へ ロック付きメモは故人アカウント管理連絡先でもアクセス不可。個人的な記録はここで守られる。
  • 個人用写真は「非表示アルバム」＋「iCloud写真同期OFF」の多層防御 iOS 18以降は非表示アルバム自体がFace IDロック。複数の層で守る。
  • 自分の持ち物を「生活の柱」と「個人用」に分類する 口座・サービス一つ一つについて「相手に存在を共有すべきか」で仕分ける。この分類作業自体が、一番大事なステップ。
  • Authenticatorアプリのバックアップコードを印刷 Google/Microsoft Authenticatorなどは本体にしか無い。バックアップコードを紙で金庫へ。
  • Googleの「アカウント無効化管理ツール」も設定 Gmail・ドライブの同等機能。一定期間未ログインで指定連絡先へ権限を渡す。
  • 相手の生年月日を正確に暗記 Apple申請で必要。お互いに自問しておく。
  • 年に一度「終活メモ」の更新日を決める 誕生日や結婚記念日。新しい口座・サービスの追随と、分類の再確認を一緒にやる日。
  • 相手の作業には干渉しない 「何を個人用にしてるの？」とは聞かない。相手の分類は相手の領域。この不可侵のルールを守ることで、仕組みが成立する。
結

この手引きは、感情や信頼の話ではなく、
権利と仕組みの話として書いた。

お互いに「見せたい」「見たい」という気持ちがあっても、
なくても、仕組みは同じように機能する。
どちらにとっても安全な、技術的な境界線をつくる——
それが、この手引きの目的。

急がなくていい。完璧でなくてもいい。
一つずつ、それぞれが自分のペースで整えていけばいい。
